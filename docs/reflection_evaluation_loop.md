# Reflection + Evaluation 循环模块文档

## 概述

Reflection + Evaluation 循环模块是咖啡推荐 AI Agent 的核心组件，用于自主评估推荐产品与用户喜好的匹配程度，并在匹配不好时自动重新推荐，确保提供最佳的推荐结果。

## 工作原理

### 循环流程

```
用户请求
    ↓
解析需求（提取偏好）
    ↓
执行推荐（search_coffee_products）
    ↓
评估匹配度（evaluate_recommendation_match）
    ↓
判断匹配质量
    ├─ 好匹配 → 输出结果
    └─ 不匹配且循环<3次 → 调整条件 → 重新推荐
        ↓
    循环=3次 → 输出结果 + 提示信息
```

### 循环次数

- **上限**：3 次
- **计数器**：每次评估后自动计数
- **退出条件**：
  1. 达到好匹配标准
  2. 循环 3 次仍未达标

## 评估维度

### 维度列表

| 维度 | 优先级 | 是否必须匹配 | 说明 |
|------|--------|-------------|------|
| 风味 | 高 | 是 | 用户最看重的维度 |
| 处理法 | 中 | 否 | 优先级中等 |
| 烘焙度 | 中 | 否 | 优先级中等 |
| 产地 | 低 | 否 | 可接受不匹配 |
| 豆种 | 低 | 否 | 可接受不匹配 |
| 价格 | 低 | 否 | 可接受不匹配 |

### 评分标准

每个维度的评分：
- **1**：匹配
- **0**：不匹配
- **None**：跳过评估（用户未提供或产品未记录）

### 匹配质量判断规则

#### 规则 1：风味必须匹配
- 如果用户提供了风味偏好
- 风味评分必须为 1
- 否则：**不是好匹配** → 需要重新推荐

#### 规则 2：处理法或烘焙度至少 1 个匹配
- 如果用户提供了处理法或烘焙度偏好
- 两者中至少 1 个评分为 1
- 如果两者都为 0：**不是好匹配** → 需要重新推荐
- 如果两者都未提供：跳过此规则

#### 规则 3：其他维度可接受不匹配
- 产地、豆种、价格可以不匹配
- 这些维度不强制要求

### 总匹配分数

```
总匹配分数 = (各维度评分之和) / (已评估维度数量)
```

范围：0.0 - 1.0

- 1.0 = 所有维度都匹配
- 0.0 = 所有维度都不匹配

## 工具函数

### evaluate_recommendation_match

评估推荐产品与用户喜好的匹配程度。

**参数**：
- `product_info` (Dict): 推荐的产品信息
- `user_preferences` (Dict): 用户的偏好信息

**返回**（JSON）：
```json
{
  "is_match": true,
  "match_score": 1.0,
  "dimension_scores": {
    "flavor": 1,
    "processing_method": 1,
    "roast_level": 1,
    "origin": 1,
    "bean_variety": null,
    "price": null
  },
  "evaluated_dimensions": ["flavor", "processing_method", "roast_level", "origin"],
  "is_good_match": true,
  "should_retry": false,
  "recommendation": "使用当前推荐",
  "reason": "产品与用户偏好匹配良好"
}
```

### 评估逻辑

#### 1. 风味维度评估

支持：
- 精确匹配："茉莉花" == "茉莉花"
- 包含匹配："花香" in "茉莉花香"
- 数组匹配：检查风味标签列表
- 多关键词："花香,果香" 拆分为多个关键词检查

#### 2. 价格维度评估

支持多种格式：
- "50-80元"：区间匹配
- "50元左右"：±20 元误差
- "50元以下"：≤ 50 元
- "50元以上"：≥ 50 元

#### 3. 其他维度评估

使用通用的字符串匹配逻辑：
- 精确匹配
- 包含匹配（不区分大小写）

## 重新推荐策略

当评估结果不是好匹配时，Agent 会：

### 策略 1：调整搜索关键词
- 放宽风味关键词的精确度
- 使用更通用的描述

### 策略 2：调整过滤条件
- 放宽非关键维度的限制
- 例如：如果不匹配的是产地，可以移除产地限制

### 策略 3：调整价格区间
- 扩大价格范围
- 移除价格限制

### 策略 4：尝试不同的搜索参数组合
- 使用部分条件重新搜索
- 调整向量搜索的权重

## 输出格式

### 正常匹配

```markdown
# 🍵 咖啡推荐结果

## 推荐产品
[产品详细信息]

## 冲煮建议
[详细的冲煮方案]

## 专业点评
[基于产品特性的专业分析]
```

### 3 次循环未达标

```markdown
# 🍵 咖啡推荐结果

## 推荐产品
[产品详细信息]

## 温馨提示
⚠️ 在咖啡产品库中未找到最佳匹配的产品，可以尝试一下这款产品。

详细原因：
[评估失败的原因说明]

## 冲煮建议
[详细的冲煮方案]

## 专业点评
[基于产品特性的专业分析]
```

## 使用示例

### 示例 1：完美匹配

**用户输入**：
```
我喜欢来自埃塞俄比亚的自然处理法咖啡豆，特别偏好茉莉花风味，想要浅烘焙的。
```

**执行流程**：
1. 搜索产品 → 找到"埃塞俄比亚耶加雪菲·花魁"
2. 评估匹配 → 风味(1)、处理法(1)、烘焙度(1)、产地(1)
3. 判断 → 好匹配
4. 输出 → 正常推荐结果

### 示例 2：需要重新推荐

**用户输入**：
```
我喜欢来自埃塞俄比亚的水洗咖啡，喜欢黑醋栗风味，想要中烘焙的。
```

**执行流程**：
1. 搜索产品 → 未找到完全匹配的产品
2. 评估匹配 → 风味不匹配（0）
3. 判断 → 不是好匹配（循环 1）
4. 重新搜索 → 调整条件
5. 评估匹配 → 仍未达到好匹配（循环 2）
6. 重新搜索 → 调整条件
7. 评估匹配 → 仍未达到好匹配（循环 3）
8. 判断 → 达到循环上限
9. 输出 → 最后一次推荐 + 提示信息

## 配置说明

### 系统提示词

在 `config/agent_llm_config.json` 中，系统提示词包含：

- **循环逻辑说明**：明确指导 Agent 执行评估循环
- **循环次数限制**：最多 3 次
- **评估规则**：详细说明匹配质量判断标准
- **输出格式**：正常匹配和未达标的输出格式

### 工具列表

    "expand_flavor_keywords",
    "check_price_match",
    "parse_user_price_range",

    "get_brewing_knowledge"
  ]

## 注意事项

### 1. 严格遵循评估规则

- 风味必须匹配（如果用户提供了）
- 处理法和烘焙度至少 1 个匹配
- 不要随意放宽关键维度

### 2. 控制循环次数

- 最多循环 3 次
- 超过后必须停止
- 输出最后一次推荐 + 提示信息

### 3. 友好的提示

- 即使推荐不完美，也要友好地说明情况
- 提供原因说明
- 给出替代建议

### 4. 评估准确性

- 确保评估工具正确实现所有规则
- 测试各种边界情况
- 验证输出结果的准确性

## 性能优化

### 1. 缓存评估结果

如果多次评估同一个产品，可以缓存评估结果，避免重复计算。

### 2. 并行评估

如果需要评估多个产品，可以并行执行评估。

### 3. 提前终止

如果某个维度已经不匹配（如风味），可以提前返回，不需要评估其他维度。

## 未来改进

### 1. 语义相似度匹配

使用 LLM 或 embedding 评估风味维度的语义相似度，而不仅仅是字符串匹配。

### 2. 动态权重

根据用户反馈调整各维度的权重。

### 3. 学习用户偏好

记录用户的接受/拒绝反馈，自动调整推荐策略。

### 4. 扩展循环次数

对于高级用户，可以增加循环次数上限。
